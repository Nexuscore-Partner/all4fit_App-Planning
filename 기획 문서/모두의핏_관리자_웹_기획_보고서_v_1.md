## 1. Executive Summary
모두의핏 관리자 웹은 사용자 앱의 시설·커뮤니티·예약·알림·콘텐츠 운영 전 과정을 **승인·품질관리·정책반영** 중심으로 지원한다. 본 보고서는 운영 목표, 핵심 사용자(운영자/모더레이터/데이터 에디터 등)별 가치, 우선 개발 범위(MVP)와 고도화 로드맵을 제시한다.

### 1.1 운영 목표(KPI)
- 데이터 품질: 시설 필수 필드 충족률 95%+, 중복 시설 병합 TAT 24h 내
- 신뢰도: 신고 처리 평균 리드타임 48h → 24h, 오탐률 < 5%
- 전환: 예약 완료 전환율 +X%, 알림 클릭률 +Y%
- 효율: 반복 업무 자동화로 운영 시간 20% 절감

---

## 2. 서비스 범위 및 가정
- 범위: 대시보드, 사용자·권한, 시설, 리뷰·신고, 커뮤니티, 예약, 지도자 매칭, 알림, CMS, 분류·메타데이터, 데이터/리포트, 시스템·정책
- 가정: 운영 조직 내 역할 구분(슈퍼관리자/운영자/모더레이터/에디터/CS/분석), 공공데이터/API 연계, RBAC 준수

---

## 3. 핵심 페르소나와 가치 제안
- **슈퍼관리자**: 정책/권한/토글 통합 제어, 리스크/감사 추적
- **운영자**: 시설·커뮤니티·리뷰/신고/예약의 승인·조치·공지·배너 운영
- **모더레이터**: 신고 심사, 제재 기준 적용, 이의신청 처리
- **데이터 에디터**: 신규 시설/정보 입력·수정, 품질 점검 대응
- **CS 담당**: 사용자 이슈 탐색, 상태 확인, 기록 중심 대응
- **분석가**: 주요 지표 탐색 및 CSV/쿼리 기반 분석

---

## 4. IA 개요(정보구조)
대시보드 → 업무 큐(승인/신고/예약 이슈)로 드릴다운. 각 도메인(시설/커뮤니티/예약 등)은 **목록-상세-행위(승인·병합·환불 등)**의 3단 레이어로 설계. 공통 도메인 설정(분류/메타)과 콘텐츠(CMS)·알림은 전역으로 연결.

---

## 5. 주요 사용자 여정(운영 플로우)
1) **시설 등록 요청 처리**: 요청 접수 → 필드 검증 → 중복 탐지/병합 → 승인(상태: APPROVED) → 앱 반영  
2) **리뷰 신고 처리**: 접수 → 증빙 확인 → 조치(블라인드/삭제/경고/정지) → 통지 → 이의신청 → 재검토  
3) **예약 이슈 처리**: 결제 실패/중복 → 원인 파악 → 재시도/취소/환불 → 사용자 통지  
4) **알림 캠페인**: 템플릿 작성 → 타겟 세그먼트 → 테스트 → 예약 발송 → 성과 리포트  
5) **커뮤니티 운영**: 방 생성 승인 → 금칙어/정책 위반 모니터링 → 제재·해지

---

## 6. 기능 요구사항 요약(모듈별)
- **대시보드**: KPI/업무 큐/시스템 상태, 위젯 커스터마이즈, 드릴다운
- **사용자·권한**: 검색/필터, 권한 변경, 제재, 감사 로그
- **시설**: CRUD, 등록요청 승인, 중복 병합, 공공 API 동기화, 품질 점검
- **리뷰·신고**: 심사/조치, 자동 필터, 이의신청, 통계
- **커뮤니티**: 방 승인·폐쇄, 공지 고정, 멤버 제재
- **예약**: 슬롯 템플릿, 취소/환불, 캘린더, 모니터링
- **지도자 매칭**: 프로필 승인, 매칭 상태 전이, 분쟁 관리
- **알림**: 템플릿·세그먼트·예약 발송·성과 측정
- **CMS**: 공지/배너/문서 버전 관리
- **분류·메타**: 종목/시설유형/지역/편의시설/태그 관리
- **데이터/리포트**: CSV/저장 쿼리/스케줄링
- **시스템·정책**: 기능 토글, 최소 앱 버전, 인덱스 재빌드, API 키, 감사 알림

---

## 7. 비기능 요구사항
- 보안: RBAC, 2FA, HTTPS, PII 최소화, 필드 마스킹, 불변 감사 로그
- 성능/가용성: 서버 페이징, 인덱스 최적화, 95p < 500ms, 오류 모니터링
- 준법/감사: 개인정보 보관·파기, 제재 기록 보존, 약관 버전 트래킹
- 호환성/접근성: 데스크톱 우선 반응형, 최신 브라우저, WCAG 일부 준수

---

## 8. 데이터 거버넌스
- 원천: 공공 API/크롤링/사용자 제안  
- 스키마: 내부 표준화 맵, 충돌 해결 규칙(로컬 우선/원천 우선)  
- 품질: 필수 필드 규칙, 좌표 허용오차, 사진 해상도/저작권  
- 로깅: 생성/수정/승인/병합/제재 이력, 데이터 계보(lineage)

---

## 9. 릴리스 전략
- **MVP**: 대시보드 요약, 시설 CRUD+등록요청, 리뷰/신고, 커뮤니티 제재, 공지/배너, 기본 리포트  
- **고도화**: 예약(슬롯/환불), 지도자 매칭, 알림 A/B·세그먼트, 자동 품질 점검, 기능 토글·버전 정책

### 9.1 마일스톤
- M1(4주): IA 고정, 핵심 목록/상세 화면, RBAC·2FA, 기본 로그  
- M2(4주): 시설 승인/병합/동기화, 리뷰/신고 워크플로우, 공지/배너  
- M3(4주): 데이터 리포트, 알림 템플릿·발송, 커뮤니티 운영  
- M4(고도화): 예약, 지도자 매칭, 자동화·지표 고도화

---

## 10. 리스크 & 완화
- 공공 데이터 변동성 → 필드 매핑 버전 관리, 회귀 테스트
- 모더레이션 과부하 → 자동 필터/우선순위 큐, SLA 알림
- 데이터 무결성(병합) → 병합 전 시뮬레이션/롤백 로그
- 보안(권한 오남용) → 세분화 RBAC, 승인 2인제(선택), 감사 경보
