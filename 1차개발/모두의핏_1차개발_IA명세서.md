# 모두의 핏 1차 개발 IA(Information Architecture) 명세서

## 📋 개요
모두의 핏 앱의 정보 구조와 네비게이션 체계를 정의한 IA 명세서

## 🏗️ 전체 IA 구조

### 1. 메인 네비게이션 구조
```
모두의 핏 앱
├── 인증 영역 (비로그인)
│   ├── 로그인/회원가입
│   └── 비밀번호 찾기
├── 메인 앱 (로그인 후)
│   ├── 홈 탭
│   ├── 커뮤니티 탭
│   ├── 마이페이지 탭
│   └── 설정
└── 역할별 전용 영역
    ├── 시설 운영자 전용
    ├── 지도자 전용
    └── 관리자 전용
```

### 2. 탭 기반 네비게이션
```
하단 탭바
├── 홈 (🏠)
├── 찜 (❤️) - 2차 개발
├── 지도 (🗺️)
├── 커뮤니티 (💬)
└── 마이페이지 (👤)
```

## 📱 화면별 IA 구조

### 1. 인증 영역 IA

#### 1.1 로그인/회원가입
```

├── 비밀번호 찾기
└── 회원가입
    ├── 사용자 타입 선택
    │   ├── 일반 사용자
    │   ├── 지도자 신청
    │   └── 시설 운영자 신청
    ├── 기본정보 입력
    │   ├── 이름 입력
    │   ├── 이메일/전화번호 입력
    │   └── 비밀번호 설정
    ├── 약관동의
    ├── 사용자 설정
    │   ├── 닉네임 설정
    │   ├── 생년월일 입력
    │   ├── 성별 선택
    │   └── 거주지역 설정
    └── 가입완료
```

#### 1.2 비밀번호 찾기

## 🔗 네비게이션 플로우

### 1. 주요 사용자 플로우

#### 1.1 일반 사용자 플로우
```
앱 실행
├── 비로그인 → 로그인/회원가입
└── 로그인됨 → 메인 앱
    ├── 홈 탭
    │   ├── 시설 검색 → 시설 상세 → 리뷰/즐겨찾기
    │   └── 운동 분야 선택 → 시설 리스트 → 시설 상세
    ├── 찜 탭 (2차 개발)
    │   └── 찜한 시설 목록 → 시설 상세
    ├── 지도 탭
    │   └── 지도에서 시설 탐색 → 시설 상세
    ├── 커뮤니티 탭
    │   ├── 게시글 목록 → 게시글 상세 → 댓글/공감
    │   └── 게시글 작성
    └── 마이페이지 탭
        ├── 프로필 수정
        ├── 리뷰 관리
        ├── 1:1 문의
        └── 설정
```

#### 1.2 시설 운영자 플로우
```
로그인 → 시설 운영자 메인
├── 시설 관리
│   └── 시설 정보 수정
├── 리뷰 관리
│   └── 리뷰 답글/신고
├── 고객 관리
│   └── 고객 소통
└── 운영 통계
```

#### 1.3 지도자 플로우
```
로그인 → 지도자 메인
├── 문의 답변
│   └── 답변 작성/관리
├── 프로필 관리
│   └── 지도자 정보 수정
└── 답변 통계
```

#### 1.4 관리자 웹 플로우
```
웹 접속 → 관리자 웹 로그인
├── 웹 대시보드
│   ├── 실시간 통계
│   ├── 알림 관리
│   └── 시스템 상태
├── 사용자 관리
│   ├── 회원/지도자/운영자 관리
│   └── 승인/제재 처리
├── 콘텐츠 관리
│   ├── 리뷰/문의/신고 관리
│   └── 콘텐츠 검토/처리
├── 시스템 관리
│   ├── 시스템 설정
│   ├── 데이터 백업/복원
│   └── 권한 관리
└── 분석 및 리포트
    ├── 사용자/서비스 분석
    └── 리포트 생성
```

### 2. 네비게이션 패턴

#### 2.1 탭 네비게이션
- **하단 탭바**: 홈, 찜(2차), 지도, 커뮤니티, 마이페이지
- **탭 전환**: 즉시 전환, 상태 유지
- **탭 표시**: 아이콘 + 라벨

#### 2.2 스택 네비게이션
- **뒤로가기**: 상단 뒤로가기 버튼
- **계층 구조**: 명확한 계층 관계
- **브레드크럼**: 필요시 경로 표시

#### 2.3 모달 네비게이션
- **로그인/회원가입**: 전체화면 모달
- **설정 화면**: 슬라이드업 모달
- **확인 다이얼로그**: 중앙 팝업

## 🎯 IA 설계 원칙

### 1. 사용자 중심 설계
- **직관적 네비게이션**: 사용자가 쉽게 이해할 수 있는 구조
- **일관성**: 전체 앱에서 일관된 네비게이션 패턴
- **효율성**: 최소한의 클릭으로 원하는 기능에 접근

### 2. 역할별 맞춤화
- **일반 사용자**: 시설 검색과 커뮤니티 중심
- **시설 운영자**: 시설 관리와 고객 관리 중심
- **지도자**: 문의 답변과 프로필 관리 중심
- **관리자**: 전체 시스템 관리 중심

### 3. 확장성 고려
- **모듈화**: 각 기능 영역의 독립성
- **확장 가능**: 2차 개발 기능 추가 시 구조 유지
- **유연성**: 사용자 요구사항 변경에 대응 가능

## 📱 화면 전환 매트릭스

### 1. 메인 탭 간 전환
```
        홈    찜    지도   커뮤니티   마이페이지
홈      -    2차    즉시   즉시      즉시
찜     2차    -     2차    2차       2차
지도   즉시   2차    -     즉시      즉시
커뮤니티 즉시  2차    즉시   -       즉시
마이페이지 즉시 2차    즉시   즉시     -
```

### 2. 상세 화면 전환
```
상위 화면 → 하위 화면
홈 → 시설 리스트 → 시설 상세
지도 → 시설 상세
커뮤니티 → 게시글 목록 → 게시글 상세
마이페이지 → 설정 → 세부 설정
```

### 3. 모달 화면 전환
```
기본 화면 → 모달 화면
로그인 메인 → 회원가입
시설 상세 → 리뷰 작성
마이페이지 → 1:1 문의
```

이 IA 명세서는 모두의 핏 앱의 정보 구조와 사용자 경험을 최적화하기 위한 설계 가이드라인을 제공합니다.
