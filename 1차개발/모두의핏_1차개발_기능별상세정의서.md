# 모두의 핏 1차 개발 기능별 상세 정의서

## 개요
모두의 핏 앱의 각 기능을 상세하게 정의한 문서입니다.

## 1. 인증 시스템

### 1.1 회원가입

#### 1.1.1 사용자 타입 선택
**설명**: 회원가입 시 일반 사용자, 지도자, 시설 운영자 중 선택

**기능 상세**:
- 화면에 3개의 선택 버튼 표시
- 각 타입별 간단한 설명 제공
- 선택 후 다음 단계로 진행

**입력 데이터**:
- userType: 'general' | 'instructor' | 'operator'

**검증 규칙**:
- 반드시 하나의 타입 선택 필수
- 선택하지 않으면 다음 단계 진행 불가

#### 1.1.2 이름 입력
**설명**: 사용자의 실명 입력

**기능 상세**:
- 한글/영문 이름 입력 가능
- 2-10자 제한
- 특수문자 제한

**입력 데이터**:
- name: string (2-10자)

**검증 규칙**:
- 필수 입력
- 2자 이상 10자 이하
- 한글, 영문, 공백만 허용
- 특수문자, 숫자 불허용

#### 1.1.3 이메일/전화번호 입력
**설명**: 계정 인증을 위한 이메일과 전화번호 입력 및 인증

**기능 상세**:
- 이메일 중복 확인
- 전화번호 SMS 인증
- 인증 완료 후 다음 단계 진행

**입력 데이터**:
- email: string (이메일 형식)
- phone: string (휴대폰 번호)
- verificationCode: string (SMS 인증번호)

**검증 규칙**:
- 이메일: RFC 5322 형식 준수
- 전화번호: 010-XXXX-XXXX 형식
- 이메일 중복 확인 필수
- SMS 인증번호 6자리 숫자
- 인증번호 유효시간 3분

**처리 로직**:
1. 이메일 중복 확인 API 호출
2. 중복되지 않으면 SMS 인증번호 발송
3. 사용자가 인증번호 입력
4. 인증번호 검증 API 호출
5. 인증 성공 시 다음 단계 진행

#### 1.1.4 비밀번호 설정
**설명**: 계정 보안을 위한 비밀번호 설정

**기능 상세**:
- 비밀번호 입력 및 확인
- 보안 강도 표시
- 비밀번호 정책 안내

**입력 데이터**:
- password: string
- confirmPassword: string

**검증 규칙**:
- 8자 이상 20자 이하
- 영문 대소문자, 숫자, 특수문자 조합
- 연속된 문자 3개 이상 불허용
- 공백 불허용
- 비밀번호 확인 일치 필수

**보안 정책**:
- bcrypt로 암호화 저장
- 최소 10회 이상 해싱
- 솔트 자동 생성

#### 1.1.5 약관동의
**설명**: 서비스 이용약관 및 개인정보처리방침 동의

**기능 상세**:
- 필수 약관 및 선택 약관 구분
- 약관 내용 팝업으로 표시
- 동의 여부 체크박스
- 위치기반 서비스 약관 별도 표시

**입력 데이터**:
- agreeTerms: boolean (필수)
- agreePrivacy: boolean (필수)
- agreeLocation: boolean (필수)
- agreeMarketing: boolean (선택)

**검증 규칙**:
- 필수 약관 동의 필수
- 위치기반 서비스 약관은 위치 서비스 이용 시 필수
- 선택 약관은 선택사항
- 약관 내용 확인 후 동의

#### 1.1.6 사용자 설정
**설명**: 개인화를 위한 사용자 정보 설정

**기능 상세**:
- 닉네임 설정 (중복 확인)
- 생년월일 입력 (년/월/일 각각 텍스트 입력)
- 성별 선택
- 거주지역 설정

**입력 데이터**:
- nickname: string (2-10자)
- birthYear: string (4자리)
- birthMonth: string (2자리)
- birthDay: string (2자리)
- gender: 'male' | 'female'
- location: string

**검증 규칙**:
- 닉네임: 2-10자, 특수문자 제한
- 닉네임 중복 확인 필수
- 생년: 1900-현재년도, 4자리 숫자
- 생월: 01-12, 2자리 숫자
- 생일: 01-31, 2자리 숫자 (월별 유효 일수 확인)
- 생년월일: 만 14세 이상
- 성별: 남성/여성 중 선택
- 거주지역: 시/도, 구/군 선택

### 1.2 로그인

#### 1.2.1 통합 로그인
**설명**: 이메일 또는 전화번호로 로그인

**기능 상세**:
- 이메일/전화번호 통합 입력
- 자동 형식 인식
- 로그인 상태 유지 옵션

**입력 데이터**:
- emailOrPhone: string
- password: string
- rememberMe: boolean

**검증 규칙**:
- 이메일 또는 전화번호 형식 확인
- 비밀번호 8자 이상
- 계정 존재 여부 확인
- 비밀번호 일치 확인

**처리 로직**:
1. 입력값 형식 검증
2. 사용자 계정 조회
3. 비밀번호 해시 비교
4. JWT 토큰 생성
5. 로그인 상태 업데이트

#### 1.2.2 카카오 로그인
**설명**: 카카오 계정을 통한 소셜 로그인

**기능 상세**:
- 카카오 SDK 연동
- 카카오 계정 정보 연동
- 자동 회원가입 처리

**처리 로직**:
1. 카카오 로그인 버튼 클릭
2. 카카오 앱 또는 웹 로그인
3. 카카오 액세스 토큰 획득
4. 서버에서 카카오 API 호출
5. 사용자 정보 조회/생성
6. JWT 토큰 발급

### 1.3 비밀번호 관리

#### 1.3.1 비밀번호 찾기
**설명**: 이름과 전화번호로 비밀번호 재설정

**기능 상세**:
- 이름 입력
- 전화번호 입력 및 SMS 인증
- 새 비밀번호 설정

**입력 데이터**:
- name: string
- phone: string
- verificationCode: string
- newPassword: string

**처리 로직**:
1. 이름과 전화번호로 계정 조회
2. SMS 인증번호 발송
3. 인증번호 검증
4. 새 비밀번호 설정
5. 기존 세션 무효화

## 2. 홈 탭

### 2.1 위치 기반 서비스

#### 2.1.1 현재 위치 표시 및 변경
**설명**: 사용자의 현재 GPS 위치 기반으로 주소 노출 및 변경 가능
**기능 상세**:
- 앱 상단에 현재 위치 주소 (`서울 마포구 서교동`) 표시
- 위치 표시 영역 클릭 시 위치 설정 화면으로 이동하여 `현재 위치` 또는 `지역 선택`을 통해 위치 변경 가능
- 변경된 위치는 앱 전반의 시설 검색 및 추천에 반영
- 우측 상단에 알림 아이콘 표시
**입력 데이터**:
- latitude: float
- longitude: float
- address: string
**검증 규칙**:
- 유효한 GPS 좌표
- 위치 권한 동의 필수
**처리 로직**:
- 앱 실행 시 GPS 권한 요청 및 현재 위치 기반 주소 조회 (Geocoding API)
- 사용자가 직접 지역 선택 시 해당 지역 주소로 업데이트
- 주소 표시 및 DB 저장

#### 2.1.2 권한 요청 팝업
**설명**: 앱 사용을 위한 필수 권한 요청
**기능 상세**:
- 앱 실행 시 권한 요청 팝업 표시
- 필수 권한과 선택 권한 구분
- 사용자가 권한 허용/거부 선택
**권한 목록**:
- [필수] 기기 및 앱 기록
- [선택] 위치
- [선택] 사진/카메라
- [선택] 알림

#### 2.1.3 위치 선택 화면
**설명**: 사용자가 수동으로 위치 변경
**기능 상세**:
- 현재 위치와 지역 선택 옵션 제공
- 시/도와 구/군을 2단계로 선택
- 선택된 위치로 앱 전반 업데이트

#### 2.1.4 시설/지도자 탭 전환
**설명**: 시설과 지도자 탭 간 전환 기능
**기능 상세**:
- 시설 탭: 운동 시설 검색 및 조회
- 지도자 탭: 운동 지도자 검색 및 조회
- 탭 전환 시 해당 콘텐츠로 화면 변경

#### 2.1.5 운동 분야 아이콘 그리드
**설명**: 8가지 운동 분야를 아이콘 그리드로 표시
**기능 상세**:
- 구기종목, 단체종목, 개인/무료, 수상종목 등 8개 카테고리
- 각 아이콘 클릭 시 해당 분야 시설 목록으로 이동
- 원형 아이콘 디자인
**카테고리**:
- 구기종목 (⚽)
- 단체종목 (🏐)
- 개인/무료 (🏃)
- 수상종목 (🏊)
- 격투/무술 (🥋)
- 피트니스 (💪)
- 동계스포츠 (⛷️)
- 골프/레저 (⛳)

### 2.2 시설 검색

#### 2.2.1 시설 검색창
**설명**: 키워드 기반 시설 검색

**기능 상세**:
- 실시간 검색어 추천
- 검색 기록 저장
- 자동완성 기능

**입력 데이터**:
- keyword: string
- category?: string
- location: LocationData

**검색 로직**:
1. 검색어 입력 시 실시간 추천
2. 검색 실행 시 다중 조건 필터링
3. 거리순/평점순/즐겨찾기순 정렬
4. 페이지네이션 적용

### 2.3 운동 분야 선택

#### 2.3.1 운동 분야 선택
**설명**: 대분류 운동 종목 선택

**기능 상세**:
- 카테고리별 아이콘 표시
- 선택 시 세부 종목으로 이동

**카테고리 구조**:

1.	구기종목
   - 축구, 야구, 풋살, 농구, 볼링, 당구, 배구

  2. 라켓 종목
  - 테니스, 배드민턴, 탁구, 스쿼시

  3. 격투 / 무술
  - 태권도, 유도, 복싱, 검도, 펜싱, 종합격투기(가라테, 킥복싱, 합기도, 유술, 씨름)

  4. 수상 종목
  - 수영, 수상스키, 다이빙, 서핑, 요트, 조정, 카누

  5. 피트니스
	⁃	헬스, 필라테스, 요가, 크로스핏, 줄넘기, 댄스스포츠

 6. 동계스포츠
  - 빙상, 스키, 썰매

 7. 골프 & 레저
  - 스크린골프, 클라이밍, 게이트볼, 그라운드 골프, 인라인, 페러글라이딩

 8. 체대입시 & 장애인
  - 체대입시, 장애인


```
운동 분야
├── 구기 (⚽)
│   ├── 축구
│   ├── 농구
│   ├── 야구
│   └── 기타 구기
├── 격투/무술술 (🥋)
│   ├── 태권도
│   ├── 유도
│   └── 복싱
├── 라켓 (🎾)
│   ├── 테니스
│   ├── 베드민턴
│   └── 탁구
└── 수영 (🏊)
    ├── 수영
    └── 다이빙
```

### 2.4 근처 인기 시설

#### 2.4.1 인기 시설 추천
**설명**: 현재 위치 기반 인기 시설 추천

**기능 상세**:
- 거리 기반 추천
- 평점 기반 추천
- 이용자 수 기반 추천

**추천 알고리즘**:
1. 현재 위치 기준 반경 내 시설
2. 평점 4.0 이상 필터링
3. 리뷰 수 10개 이상 필터링
4. 최근 1개월 내 이용자 수 고려
5. 거리순으로 상위 10개 추천

### 2.5 시설 상세

#### 2.5.1 시설 상세 페이지 진입 및 기본 정보
**설명**: 선택된 시설의 상세 정보를 보여주는 페이지. 상단에 기본 정보와 하단에 탭 메뉴를 통해 세부 정보를 제공.
**기능 상세**:
- 시설명, 현재 위치 기준 거리 표시
- 공유하기, 즐겨찾기(찜) 기능
- 시설 이미지 슬라이드
- 평점 및 리뷰 수 표시
- 주소 표시
- 상담 바로가기 버튼
- 하단 탭 메뉴 (시설정보, 지도자, 예약, 후기, 요금)
**입력 데이터**:
- facilityId: string
**검증 규칙**:
- facilityId: 필수, 유효한 시설 ID
**처리 로직**:
- facilityId를 통해 시설 기본 정보 조회
- 조회된 정보를 화면 상단에 표시
- 기본 탭은 '시설정보'로 설정

#### 2.5.2 시설정보 탭
**설명**: 시설의 운영 종목, 편의 시설, 상세 위치 정보를 제공
**기능 상세**:
- 시설 운영 종목 리스트 표시
- 편의 시설 (주차, 샤워실 등) 체크박스 형태로 표시
- 상세 위치를 보여주는 지도 스니펫
- 문의하기 버튼
**입력 데이터**:
- facilityId: string (탭 전환 시 내부적으로 사용)
**검증 규칙**:
- facilityId: 필수, 유효한 시설 ID
**처리 로직**:
- facilityId에 해당하는 시설의 운영 종목, 편의 시설, 위치 정보 조회
- 조회된 정보를 화면에 표시


#### 2.5.3 지도자 탭 (1차 개발: 조회만 가능)
**설명**: 해당 시설에 소속된 지도자 리스트를 제공
**기능 상세**:
- 지도자 프로필 이미지, 한줄 소개, 이름 표시
- 각 지도자 클릭 시 상세 페이지로 이동 (2차 개발)
**입력 데이터**:
- facilityId: string (탭 전환 시 내부적으로 사용)
**검증 규칙**:
- facilityId: 필수, 유효한 시설 ID
**처리 로직**:
- facilityId에 해당하는 시설의 지도자 목록 조회
- 조회된 지도자 정보를 리스트 형태로 표시

#### 2.5.4 예약 탭 (외부 홈페이지 연동)

## 🚀 10. 2차 개발 기능 (예약 시스템)

### 10.1 시설 예약 기능 (2차 개발)
**설명**: 사용자가 시설의 특정 시간대를 예약하고 관리하는 기능 (UI에는 '예약 가능' 태그로 표시되나, 실제 기능은 2차 개발)

**기능 상세**:
- 시설 상세 페이지에서 '예약하기' 버튼 클릭 시 예약 캘린더 화면으로 이동
- 예약 캘린더에서 예약 가능한 시간대 조회 및 선택
- 선택한 시간대에 대한 예약 신청 및 결제 (결제는 별도 연동)
- 마이페이지에서 예약 내역 조회 및 상태 (대기/확정/취소) 확인
- 예약 확정/취소/변경 시 사용자에게 푸시 알림 발송

**입력 데이터**:
- facilityId: string
- userId: string
- reservationDate: Date
- reservationTime: string
- paymentInfo: object (결제 정보)

**검증 규칙**:
- 유효한 시설 ID, 사용자 ID
- 예약 가능 시간대 확인
- 결제 정보 유효성 검증

**처리 로직**:
- 예약 요청 시 예약 DB에 저장 및 시설 운영자에게 알림
- 시설 운영자의 승인/거절에 따라 예약 상태 업데이트
- 결제 시스템 연동


**비고**: UI에 '예약 가능' 태그는 1차 개발 시 표시되지만, 실제 예약 신청 및 관리 기능은 2차 개발 범위에 포함됩니다.

**검증 규칙**:
